Goal of website

Test your memory with cards game

The memory game is a game of which you are supposed to recall and match identical cards by flipping them to find pairs. It's a challenging game that helps with sharpening memory and concentration abilities.

The timer section calculates how much time you spent to complete the game

The cards section calculates how many cards you have collected

When pausing the game, you won't have acces to cards till the game is resumed again

When resuming the game. The timer will continue the track and you can continue the game

What are the difficulties endured while coding?

At first, I assigned identical cards the same ID for tracking. But this affected the manipulation of cards, which made the interraction works only with the first cards.
Solution: Using a new attribute instead of their id, to shuffle the cards and keep track of identic ones

The timer wasn't functionning as it should when the "start game" button is clicked more than once. Apparently, each time the button is clicked, a new timer is generated, which makes more than one timer manipulating the timer section.
Solution: Using a variable to keep tracking the interval of timer, to know when to use a "clearInterval" or "setInterval"

When the game is pause, the user can get to play even if the timer is paused, which is considered as cheating.
Solution: Disabeling the interraction between user and the games section, untill the resume button is clicked

How is the quote generated?

The quotes are generated using an API. the data gets fetched. After that, the quote and its author gets extracted and saved in the quote section. This API has a limit requests by time. So it's normal to get an error after trying too many times. You can come back later and it will work again.
